# Reti, web, posta elettronica e protocolli per concorsi

## Sintesi

Questa fonte consolida gli standard essenziali per rispondere alle domande su browser, URL, web, posta elettronica, DNS, TCP/IP, LAN/WAN, client/server, router, switch, modem e servizi Internet. Gli standard IETF sono fonti tecniche primarie; per i concorsi non vanno studiati integralmente, ma usati per definizioni corrette e per evitare semplificazioni sbagliate.

## Nuclei ad alta frequenza

- URL/URI: schema, host, path, query, frammento; differenza tra indirizzo web e pagina.
- HTTP/HTTPS: richiesta, risposta, risorsa, metodo, status code in senso generale; HTTPS come HTTP su canale protetto.
- TCP/IP: pacchetti, indirizzi, porte, trasporto affidabile; differenza tra IPv4 e IPv6 a livello concettuale.
- DNS: risoluzione dei nomi di dominio in indirizzi, gerarchia, record.
- Email: invio tramite SMTP, caselle, allegati, campi mittente/destinatario/cc/ccn; PEC come sistema con ricevute e valore legale nel contesto italiano.
- Reti: LAN, WAN, Internet, intranet, client/server, router, switch, modem, firewall.

## Note concorsuali

Le domande chiedono raramente dettagli da amministratore di rete; chiedono invece definizioni, riconoscimento di acronimi e relazioni funzionali. Il capitolo deve insegnare "a cosa serve" un protocollo prima di spiegare "come funziona".

## Note di review

- Integrare con esempi PA: accesso a un portale, invio PEC, autenticazione, pagamento online, chiamata API tra amministrazioni.
- Evitare dettagli eccessivi degli RFC nel testo finale: sono fonte di validazione, non programma da memorizzare integralmente.
