# Capitolo 42 - Banca dati ufficiale: studiarla senza memorizzare male

La banca dati ufficiale sembra una scorciatoia.

Hai le domande. Hai le risposte. Sai che cosa puo uscire. Il problema, almeno in apparenza, e' risolto.

In realta, molti candidati peggiorano proprio quando esce la banca dati. Entrano in modalita accumulo: mille quiz al giorno, screenshot, file duplicati, app, gruppi, ripassi casuali, classifiche di percentuale. Dopo pochi giorni riconoscono molte domande, ma non sanno piu se stanno capendo o solo ricordando l'ordine delle opzioni.

La banca dati ufficiale e' potente solo se la tratti come uno strumento di procedura, non come una giostra di domande.

Questo capitolo ti insegna a usarla senza memorizzare male.

La regola e' semplice:

> la banca dati ufficiale non si consuma; si governa.

## Obiettivo del capitolo

Alla fine del capitolo saprai:

- distinguere una banca dati ufficiale da una raccolta di allenamento;
- inserirla nel Bando Decoder;
- costruire una mappa di copertura;
- studiare le domande senza fissarti su lettera, posizione o riconoscimento visivo;
- classificare ogni domanda con tag operativi;
- trasformare errori e incertezze in ripassi distribuiti;
- impostare simulazioni coerenti con bando, soglie, tempo e penalita;
- gestire aggiornamenti, errata corrige e versioni;
- convertire i quiz piu importanti in spiegazioni utili anche per scritto e orale.

Questo capitolo non sostituisce il capitolo sulla prova a quiz. Lo approfondisce in un punto specifico: che cosa fare quando hai davanti una banca dati ufficiale e devi renderla un vantaggio reale.

## Prima domanda: e' davvero ufficiale?

Non tutte le banche dati sono uguali.

Nel linguaggio dei candidati, "banca dati" puo voler dire molte cose: file pubblicato dall'amministrazione, raccolta commerciale, quiz di un corso, archivio di vecchi concorsi, elenco condiviso in un gruppo, domande generate da un software.

Per il Metodo BANDO la distinzione e' netta:

| Tipo di banca dati | Come trattarla |
|---|---|
| Banca dati ufficiale della procedura | Materiale prioritario. Va verificata, coperta, aggiornata, ripetuta e simulata. |
| Banca dati ufficiale di altra procedura | Materiale utile ma non decisivo. Aiuta su nuclei simili, ma non definisce il tuo programma. |
| Banca dati commerciale o di corso | Allenamento. Non sostituisce bando, avvisi e fonti ufficiali. |
| Raccolta trovata online | Materiale da usare con prudenza. Puo contenere errori, versioni superate o domande non pertinenti. |
| Quiz generati o rielaborati | Strumento di richiamo, non fonte della procedura. |

Prima di iniziare a studiare, devi compilare questa scheda.

| Controllo | Risposta |
|---|---|
| Il bando o un avviso prevede una banca dati? | |
| Dove e' pubblicata? | |
| Qual e' la data di pubblicazione? | |
| Esiste un codice, un nome file o una versione? | |
| Sono previste esclusioni di domande? | |
| Sono uscite correzioni o errata corrige? | |
| La banca dati riguarda preselettiva, scritto o altra fase? | |
| Il bando indica numero domande, tempo, soglia e penalita? | |

Se non sai rispondere a queste domande, non sei ancora pronto per fare quiz. Sei ancora nella fase B: bando e avvisi.

## La mappa BANDO della banca dati

La banca dati ufficiale va inserita dentro il Metodo BANDO.

| Fase BANDO | Domanda guida | Output pratico |
|---|---|---|
| **B - Bando** | Che cosa dice la procedura sulla banca dati? | Scheda tecnica: fonte, data, versione, prova, tempo, punteggio, soglia, penalita. |
| **A - Aree** | Quali materie e nuclei contiene? | Mappa materie/nuclei/domande. |
| **N - Nuclei** | Quali blocchi ricorrono di piu o pesano di piu? | Lista priorita: alta resa, media resa, recupero. |
| **D - Domande** | Quali quesiti sono sicuri, incerti, lenti o sbagliati? | Registro tag domanda e Diario degli errori. |
| **O - Output** | Riesco a rispondere nel tempo e a spiegare il perche? | Simulazioni, micro-spiegazioni, fascicolo finale. |

Questa tabella impedisce l'errore piu frequente: aprire la banca dati e partire dalla prima domanda senza sapere che cosa stai misurando.

Una banca dati non e' una fila di quiz. E' una mappa della prova.

## Il protocollo in quattro fasi

Con banca dati ufficiale, il lavoro procede in quattro fasi.

1. Mappare e coprire.
2. Capire e classificare.
3. Ripetere e distribuire.
4. Simulare.

Se inverti l'ordine, perdi controllo. La simulazione fatta troppo presto produce ansia e percentuali inutili. La ripetizione senza classificazione produce memoria fragile. La copertura senza mappa produce confusione.

### Fase 1 - Mappare e coprire

Il primo giro non serve a dimostrare che sei bravo.

Serve a sapere che cosa c'e' dentro.

Obiettivo della fase 1:

- passare tutta la banca dati almeno una volta;
- dividere le domande per materia o nucleo;
- segnare quelle gia sicure;
- isolare quelle non comprese;
- identificare domande lunghe, ambigue, lente o ad alta probabilita di errore;
- evitare simulazioni complete premature.

Durante il primo giro non devi cercare la perfezione. Devi costruire una fotografia affidabile.

Usa una tabella semplice.

| Nucleo | Numero domande | Sicure | Incerte | Sbagliate | Lente | Note |
|---|---:|---:|---:|---:|---:|---|
| Procedimento amministrativo | | | | | | |
| Accesso e trasparenza | | | | | | |
| Pubblico impiego | | | | | | |
| Enti locali | | | | | | |
| Informatica | | | | | | |
| Logica | | | | | | |

Se la banca dati e' molto ampia, puoi lavorare per blocchi. Ma ogni blocco deve lasciare una traccia: che cosa ho visto, che cosa so, che cosa devo riprendere.

### Fase 2 - Capire e classificare

Nel secondo giro non devi "rifare tutto". Devi capire che cosa e' successo.

Ogni domanda non sicura riceve un tag.

| Tag | Significato | Azione |
|---|---|---|
| Sicura | Rispondo correttamente e so spiegare il perche. | Ripasso leggero. |
| Incerta | Rispondo, ma con dubbio reale. | Rivedere concetto e distrattori. |
| Sbagliata | Scelgo risposta errata. | Diario degli errori. |
| Lenta | Rispondo bene ma impiego troppo tempo. | Allenamento tempo e riconoscimento nucleo. |
| Indovinata | Risposta corretta senza controllo del ragionamento. | Trattare come errore potenziale. |
| Da spiegare | Domanda collegata a concetto utile per orale o scritto. | Micro-spiegazione in 60-90 secondi. |

La domanda "indovinata" e' pericolosa. La percentuale la conta come corretta, ma il concorso puo punirla alla domanda successiva, quando cambia un distrattore o il testo e' leggermente diverso.

Nel Metodo BANDO, una domanda e' davvero acquisita solo quando soddisfa tre condizioni:

- scegli la risposta corretta;
- sai spiegare la regola o il concetto;
- sai dire perche almeno un distrattore e' sbagliato.

### Fase 3 - Ripetere e distribuire

Ripetere non significa rifare sempre tutto da capo.

Significa riportare davanti a te le domande giuste al momento giusto.

Le domande sicure tornano poco. Le domande sbagliate, lente, indovinate o da spiegare tornano spesso. Questo riduce il rumore e concentra energia dove il punteggio puo cambiare.

Un criterio pratico:

| Stato domanda | Frequenza di ritorno |
|---|---|
| Sicura due volte consecutive | Ripasso leggero ogni 7-10 giorni o nel fascicolo finale. |
| Incerta | Ritorno entro 48-72 ore. |
| Sbagliata | Ritorno entro 24-48 ore, dopo correzione concettuale. |
| Lenta | Ritorno in sessione cronometrata. |
| Indovinata | Ritorno come se fosse sbagliata. |
| Da spiegare | Ritorno con micro-esposizione orale o scritta. |

Il ripasso distribuito serve a evitare la falsa sicurezza del giorno stesso. Una domanda rivista subito sembra facile. Una domanda rivista dopo tre giorni dice molto di piu.

### Fase 4 - Simulare

Solo alla fine la banca dati diventa prova.

Simulare significa riprodurre il piu possibile le condizioni previste dal bando:

- numero di quesiti;
- tempo disponibile;
- modalita di selezione delle domande;
- punteggio;
- soglia;
- penalita, se prevista;
- divieto di pause o consultazioni;
- correzione finale;
- decisione di recupero.

La simulazione non serve a generare una percentuale da pubblicare in un gruppo. Serve a decidere.

Dopo ogni simulazione devi scrivere:

| Dato | Risposta |
|---|---|
| Punteggio stimato | |
| Distanza dalla soglia | |
| Domande sbagliate per materia | |
| Domande sbagliate per causa | |
| Domande lente | |
| Rischio principale | |
| Azione delle prossime 24 ore | |
| Azione dei prossimi 3 giorni | |

Se una simulazione non produce almeno una decisione, e' solo intrattenimento con timer.

## Come studiare una singola domanda

Il modo in cui studi una domanda determina la qualita della memoria.

Metodo debole:

1. leggo la domanda;
2. guardo le opzioni;
3. scelgo la risposta;
4. controllo se e' giusta;
5. passo oltre.

Metodo BANDO:

1. leggo la domanda e identifico il nucleo;
2. provo a rispondere prima di farmi guidare dalle opzioni;
3. scelgo la risposta;
4. spiego in una frase perche e' corretta;
5. individuo almeno un distrattore e spiego perche e' sbagliato;
6. assegno un tag;
7. se serve, trasformo la domanda in una scheda o in una micro-spiegazione.

Esempio di scheda:

| Campo | Contenuto |
|---|---|
| ID domanda | |
| Materia | |
| Nucleo | |
| Risposta corretta | |
| Perche e' corretta | |
| Distrattore pericoloso | |
| Perche mi ha attirato | |
| Tipo errore | |
| Prossimo ritorno | |

Questa scheda non va compilata per tutte le domande. Sarebbe insostenibile. Va usata per domande sbagliate, indovinate, lente o concettualmente centrali.

## Memorizzazione utile e memorizzazione cattiva

Con una banca dati ufficiale, un certo grado di memoria e' inevitabile. Non e' un problema.

Il problema e' memorizzare la cosa sbagliata.

| Memorizzazione cattiva | Memorizzazione utile |
|---|---|
| Ricordare che la risposta era "la C". | Ricordare la regola che rende corretta quella risposta. |
| Riconoscere la posizione dell'opzione. | Riconoscere il nucleo della domanda. |
| Ripetere l'intero testo a vista. | Saper sintetizzare il concetto in una frase. |
| Fidarsi del colore o dell'interfaccia dell'app. | Saper rispondere anche se l'ordine cambia. |
| Saltare la correzione perche la percentuale e' alta. | Studiare le domande corrette per caso. |

Una prova puo presentare le opzioni in ordine diverso, puo estrarre domande in modo non prevedibile, puo inserire istruzioni specifiche o puo essere somministrata in un ambiente diverso da quello in cui ti sei allenato.

Per questo devi allenare il contenuto, non lo schermo.

## Il Diario degli errori per banca dati

Il Diario degli errori cambia forma quando lavori su banca dati ufficiale.

Non basta scrivere "sbagliata". Devi sapere perche.

| Tipo errore | Segnale | Correzione |
|---|---|---|
| Memoria | Conoscevo il tema, ma non ricordavo il dato o la regola. | Richiamo attivo e ripasso distribuito. |
| Concetto | Non avevo capito la distinzione di fondo. | Tornare alla fonte o alla scheda teorica. |
| Lettura | Ho letto male una negazione, un "non", un "sempre", un "salvo". | Allenamento su parole-spia e rilettura finale. |
| Distrattore | Un'opzione plausibile mi ha attirato. | Confronto tra risposta corretta e risposta scelta. |
| Tempo | Sapevo rispondere, ma troppo lentamente. | Sessioni brevi cronometrate. |
| Strategia | Ho risposto quando dovevo saltare o ho saltato quando dovevo rispondere. | Regola personale su rischio e penalita. |

Esempio:

| Domanda | Errore | Causa vera | Azione |
|---|---|---|---|
| Accesso documentale e accesso civico | Scelto istituto sbagliato | Concetto | Rifare scheda comparativa e 10 quiz mirati. |
| Termine procedimentale | Risposta corretta ma lenta | Tempo | 15 domande cronometrate su procedimento. |
| Quesito con "non" | Sbagliata | Lettura | Evidenziare parole-spia prima di rispondere. |

Il Diario e' utile solo se modifica il piano. Se oggi sbagli tre domande sullo stesso nucleo, domani quel nucleo entra nel piano. Se dopo una settimana l'errore resta vivo, entra nel fascicolo finale.

## Il piano 21/14/7 giorni

Non sempre hai mesi. Spesso la banca dati viene pubblicata quando il calendario e' gia vicino.

Usa questo schema come modello adattabile.

### Se mancano 21 giorni

| Periodo | Obiettivo | Azione |
|---|---|---|
| Giorni 21-15 | Copertura completa | Primo giro su tutta la banca dati, mappa materie, tag iniziali. |
| Giorni 14-8 | Consolidamento | Secondo giro su sbagliate, incerte, lente, indovinate. |
| Giorni 7-4 | Simulazioni controllate | Prove con timer, correzione, recupero per nuclei. |
| Giorni 3-1 | Fascicolo finale | Errori vivi, parole-spia, domande lente, logistica prova. |

### Se mancano 14 giorni

Riduci ambizione e aumenta controllo.

| Periodo | Obiettivo | Azione |
|---|---|---|
| Giorni 14-10 | Copertura rapida | Passaggio completo ma sobrio, tag essenziali. |
| Giorni 9-5 | Errori ad alta resa | Solo sbagliate, incerte, lente, nuclei ricorrenti. |
| Giorni 4-2 | Simulazioni | Timer, punteggio, soglia, penalita. |
| Ultimo giorno | Stabilita | Niente inseguimenti: fascicolo finale e routine. |

### Se mancano 7 giorni

Non provare a fare tutto perfettamente. Devi evitare dispersione.

| Priorita | Azione |
|---|---|
| Verifica versione ufficiale | Controlla fonte, data, avvisi e correzioni. |
| Copertura selettiva | Passa i blocchi piu pesanti o piu deboli. |
| Errori ricorrenti | Lavora solo su cio che puo cambiare punteggio. |
| Simulazione breve | Misura tempo e soglia senza distruggere energia. |
| Fascicolo finale | 20-50 domande o schede che non devono piu sorprenderti. |

Quando il tempo e' poco, il candidato forte non cerca completezza estetica. Cerca riduzione del rischio.

## Aggiornamenti, correzioni e versioni

La banca dati ufficiale puo essere aggiornata, corretta, integrata o accompagnata da avvisi.

Per questo devi conservare una scheda versione.

| Campo | Dato |
|---|---|
| Fonte ufficiale | |
| Data download | |
| Nome file o codice | |
| Numero domande | |
| Avvisi collegati | |
| Errata corrige | |
| Ultimo controllo | |
| Differenze rispetto alla versione precedente | |

Non affidarti al "me l'hanno detto nel gruppo". Il gruppo puo segnalare un avviso, ma la decisione si prende sulla fonte ufficiale.

Se esce una correzione:

1. salva o annota la nuova versione;
2. identifica le domande coinvolte;
3. aggiorna il tuo registro;
4. ripassa solo cio che cambia;
5. evita di riaprire tutto il piano se non serve.

La gestione delle versioni non e' burocrazia. E' protezione del punteggio.

## Quando la banca dati diventa trappola

La banca dati aiuta, ma puo farti perdere lucidita.

Ecco i segnali di allarme:

- fai molte domande ma non sai quali nuclei stai coprendo;
- ripeti solo le domande che gia conosci;
- la percentuale sale, ma nelle simulazioni con timer scendi;
- riconosci la risposta dal posto, non dal contenuto;
- confondi banca ufficiale e quiz non ufficiali;
- studi fino a tarda notte per inseguire numeri;
- non sai spiegare le domande che fai bene;
- gli errori si ripetono ma non cambiano il piano.

Se compaiono tre di questi segnali, fermati per una sessione e rimetti ordine:

1. aggiorna la mappa;
2. conta errori per nucleo;
3. scegli tre priorita;
4. riduci il volume;
5. riparti da correzione e simulazione breve.

## Dalla domanda alla spiegazione

Una banca dati ufficiale puo aiutarti anche se dopo il quiz c'e' uno scritto o un orale.

Ogni domanda importante puo diventare una micro-spiegazione.

Formula:

1. "La domanda riguarda..."
2. "La regola e'..."
3. "La risposta corretta e'..."
4. "Il distrattore e' sbagliato perche..."
5. "In prova devo ricordare..."

Esempio astratto:

| Passaggio | Contenuto |
|---|---|
| Nucleo | Differenza tra due istituti amministrativi. |
| Regola | I due istituti hanno presupposti, effetti e procedura diversi. |
| Risposta | Scelgo l'opzione coerente con il presupposto indicato. |
| Distrattore | E' plausibile per nome, ma appartiene all'altro istituto. |
| Promemoria | Leggere prima il presupposto, poi l'effetto. |

Questa trasformazione e' importante per due motivi.

Primo: rende piu solida la memoria del quiz.

Secondo: evita che la banca dati ti renda muto davanti a una domanda orale. Se sai solo riconoscere una risposta multipla, non hai ancora trasformato il quiz in competenza esprimibile.

## Caso guidato: Luca e la banca dati da 3.000 quiz

Luca prepara una prova preselettiva con banca dati ufficiale. Il file contiene 3.000 quesiti. Mancano 18 giorni.

Il primo impulso e' fare piu quiz possibile ogni giorno.

Dopo due giorni ha questi dati:

| Indicatore | Dato |
|---|---:|
| Quiz svolti | 720 |
| Percentuale media | 74% |
| Domande segnate | 0 |
| Errori classificati | 0 |
| Simulazioni complete | 3 |
| Decisioni operative | 0 |

Sembra lavoro. In realta e' rumore.

Luca cambia metodo.

### Giorni 18-13: copertura controllata

Divide la banca dati per materia. Non cerca subito la percentuale massima. Vuole sapere dove cade.

| Materia | Domande viste | Errore principale |
|---|---:|---|
| Diritto amministrativo | 420 | Confusione tra istituti vicini |
| Pubblico impiego | 260 | Memoria di regole specifiche |
| Trasparenza/privacy | 220 | Distrattori lessicali |
| Informatica | 180 | Sicurezza e termini tecnici |
| Logica | 160 | Tempo |

### Giorni 12-8: Diario degli errori

Non rifare tutto. Lavora sui tag.

| Tag | Numero | Azione |
|---|---:|---|
| Sbagliate | 138 | Schede e ripasso entro 48 ore |
| Incerte | 211 | Secondo giro con spiegazione |
| Lente | 74 | Sessioni da 20 domande cronometrate |
| Indovinate | 96 | Trattate come rischio |
| Da spiegare | 42 | Micro-esposizione |

La percentuale iniziale non era il dato piu importante. Il dato piu importante era la composizione dell'errore.

### Giorni 7-4: simulazioni

Luca fa tre simulazioni complete. Dopo ogni simulazione scrive una decisione.

| Simulazione | Esito | Decisione |
|---|---|---|
| 1 | Sopra soglia, molti errori in trasparenza | Ripasso mirato su accesso, pubblicazione, dati personali |
| 2 | Buon punteggio, tempo stretto in logica | Allenamento blocchi logica con timer |
| 3 | Stabile, errori residui su domande lette male | Routine parole-spia e rilettura finale |

### Giorni 3-1: fascicolo finale

Luca non riapre tutto. Costruisce un fascicolo finale:

- 35 domande sbagliate piu di una volta;
- 20 domande lente;
- 15 distrattori ricorrenti;
- 10 micro-spiegazioni;
- scheda versione banca dati;
- istruzioni prova;
- routine logistica.

Arriva alla prova senza la sensazione di sapere tutto. Ma sa che cosa deve controllare.

Questo e' il punto.

## Se non esiste banca dati ufficiale

Questo capitolo non va applicato meccanicamente alle prove senza banca dati ufficiale.

Se la banca dati non esiste, il metodo cambia:

- il bando resta la fonte principale;
- le materie e i nuclei guidano il piano;
- i quiz servono per allenare tipologie;
- le raccolte non ufficiali non diventano programma;
- la correzione deve tornare alla teoria;
- le simulazioni devono essere costruite su fonti affidabili e coerenti.

La banca dati ufficiale permette copertura. Senza banca dati, devi costruire probabilita ragionevole.

Confondere i due scenari e' un errore serio.

## Da sapere in 5 righe

1. La banca dati ufficiale e' materiale di procedura: va verificata su bando e avvisi.
2. Prima si copre, poi si classifica, poi si ripete, poi si simula.
3. Una risposta corretta per caso e' un errore nascosto.
4. Il Diario degli errori deve distinguere memoria, concetto, lettura, distrattore, tempo e strategia.
5. La simulazione vale solo se produce una decisione per il piano.

## Domanda da commissario

**Domanda:** se il concorso pubblica una banca dati ufficiale, e' sufficiente memorizzare tutte le risposte?

**Risposta efficace:** no. La banca dati ufficiale va coperta integralmente quando possibile, ma la memorizzazione meccanica della lettera o della posizione e' fragile. Il candidato deve verificare fonte e versione, classificare domande sicure, incerte, sbagliate e lente, correggere gli errori per causa, ripetere in modo distribuito e simulare secondo tempo, soglia, punteggio e penalita indicati dal bando. La risposta e' davvero acquisita solo se il candidato sa anche spiegare perche e' corretta e perche i distrattori sono sbagliati.

## Domanda-trappola

**Domanda:** ho una banca dati commerciale molto ampia. Posso trattarla come banca dati ufficiale?

**Risposta:** no. Una banca dati commerciale puo essere utile per allenamento, ma non sostituisce la banca dati ufficiale della procedura, se prevista, ne sostituisce bando e avvisi. Puo aiutarti a esercitare nuclei e tipologie, ma non deve decidere da sola priorita, copertura e simulazione finale.

## Errore tipico

L'errore tipico e' misurare il lavoro con il numero di quiz svolti.

"Ho fatto 1.200 domande" non significa molto.

Conta di piu sapere:

- quante domande hai coperto;
- quali nuclei restano deboli;
- quali errori si ripetono;
- quali domande sono lente;
- quale punteggio ottieni con timer e regole reali;
- che cosa cambia nel piano dopo la correzione.

Il volume senza diagnosi crea stanchezza, non preparazione.

## Mini-esercizio

Prendi una banca dati ufficiale o, se non ne hai una, un blocco di 50 quiz coerente con il tuo bando. Lavora cosi:

1. segna fonte, data e versione;
2. dividi le 50 domande per materia;
3. rispondi senza timer;
4. assegna un tag a ogni domanda;
5. scegli solo 10 domande non sicure;
6. per ciascuna scrivi perche la risposta corretta e' corretta;
7. indica il distrattore piu pericoloso;
8. rifai le 10 domande dopo 48 ore;
9. misura quante sono diventate davvero sicure;
10. scrivi una decisione per il piano dei prossimi tre giorni.

Se alla fine hai solo una percentuale, l'esercizio e' incompleto. Se hai una decisione, il metodo sta funzionando.
