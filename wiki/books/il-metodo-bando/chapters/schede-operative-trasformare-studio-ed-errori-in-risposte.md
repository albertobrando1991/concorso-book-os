# Capitolo 35 - Schede operative: trasformare studio ed errori in risposte

Molti candidati producono appunti.

Pochi producono strumenti.

La differenza si vede quando arriva una domanda: chi ha appunti cerca dove aveva scritto il concetto; chi ha strumenti sa gia come trasformarlo in risposta.

Una scheda operativa serve a questo.

Non e' un riassunto piu ordinato.

Non e' una pagina copiata dal manuale.

Non e' una raccolta di frasi da imparare a memoria.

E' una pagina breve che collega quattro cose:

- una domanda;
- un nucleo;
- un errore da evitare;
- un output da produrre.

Se manca uno di questi elementi, la scheda rischia di diventare archivio passivo.

Questo capitolo ti insegna a costruire schede che lavorano per te: schede di concetto, norma, procedimento, confronto, domanda orale, errore e caso.

## Obiettivo del capitolo

Alla fine del capitolo saprai:

- distinguere appunto, riassunto, flashcard e scheda operativa;
- costruire schede brevi ma utili per quiz, scritto, orale e casi;
- trasformare una fonte in domanda e risposta;
- usare il Diario degli errori per decidere quali schede creare;
- evitare schede troppo lunghe, decorative o non verificabili;
- aggiornare una scheda dopo simulazioni, nuovi errori o nuovi bandi;
- usare le schede come capitale di studio riutilizzabile;
- preparare una risposta concorsuale partendo da una scheda, non dal manuale intero.

La regola e' questa:

> una scheda non deve contenere tutto. Deve farti produrre qualcosa.

## La mappa BANDO della scheda operativa

Ogni scheda deve stare dentro il Metodo BANDO.

| Fase | Domanda | Effetto sulla scheda |
|---|---|---|
| B - Bando | dove compare questo tema nel programma? | decide se la scheda serve davvero |
| A - Aree | a quale materia o profilo appartiene? | collega core e modulo specifico |
| N - Nuclei | qual e' il concetto ad alta resa? | evita riassunti troppo larghi |
| D - Diario | quale errore deve prevenire? | rende la scheda correttiva |
| O - Output | che cosa devo saper produrre? | quiz, risposta breve, orale, caso o confronto |

La scheda non parte dal desiderio di ordinare.

Parte da una domanda operativa:

> dove usero questa pagina?

Se non sai rispondere, non creare ancora la scheda.

## Appunto, riassunto, flashcard, scheda

Quattro strumenti, quattro funzioni diverse.

| Strumento | Funzione | Rischio |
|---|---|---|
| Appunto | registra cio che hai capito o sentito | resta disordinato |
| Riassunto | riduce un testo lungo | puo diventare lettura passiva |
| Flashcard | forza richiamo rapido | puo isolare il concetto |
| Scheda operativa | trasforma conoscenza in output | richiede selezione |

Non devi eliminare appunti, riassunti o flashcard.

Devi sapere quando usarli.

L'appunto nasce durante lo studio.

Il riassunto riduce volume.

La flashcard allena memoria e riconoscimento.

La scheda operativa prepara una prestazione.

In un concorso, la prestazione puo essere:

- scegliere una risposta;
- spiegare un concetto;
- distinguere due istituti;
- risolvere un caso;
- rispondere a una domanda orale;
- evitare un errore ricorrente.

La scheda e' il ponte tra studio e prestazione.

## La struttura minima

Una scheda operativa deve stare, se possibile, in una pagina.

Non per estetica.

Perche una pagina costringe a scegliere.

Usa questa struttura.

| Campo | Cosa scrivere |
|---|---|
| Domanda guida | la domanda a cui la scheda deve rispondere |
| Nucleo | definizione o idea centrale in poche righe |
| Fonte | bando, manuale, source note, norma o capitolo |
| Uso in prova | quiz, scritto, orale, caso, situazionale, ripasso |
| Esempio | situazione concreta o mini-applicazione |
| Errore tipico | confusione da evitare |
| Verifica | domanda, flashcard, mini-risposta o simulazione |

Se una scheda non ha domanda guida, diventa un pezzo di manuale.

Se non ha errore tipico, non corregge.

Se non ha uso in prova, non sai quando riaprirla.

## Regola 1-1-1

Per evitare schede infinite, usa la regola 1-1-1:

- una scheda;
- una domanda principale;
- un uso principale.

Esempio:

Non scrivere una scheda su "trasparenza, anticorruzione e privacy".

E' troppo vasta.

Scrivi invece:

- "Qual e' la differenza tra accesso civico semplice e generalizzato?";
- "Perche trasparenza non significa pubblicare tutto?";
- "Come rispondo a un caso con dati personali e richiesta di accesso?";

Ogni scheda deve avere un bersaglio.

Le schede larghe fanno sentire ordinati.

Le schede mirate fanno rispondere meglio.

## Scheda di concetto

La scheda di concetto serve per definizioni, funzioni e parole chiave.

Usala quando un tema torna spesso e devi spiegarlo in modo rapido.

### Modello

| Campo | Risposta |
|---|---|
| Domanda guida | Che cos'e'...? |
| Definizione breve | |
| Funzione | a cosa serve nella PA o nella prova |
| Parole chiave | 3-5 termini essenziali |
| Esempio | |
| Errore tipico | |
| Domanda orale | |

### Come usarla

Leggi la domanda guida, copri la risposta e prova a spiegare il concetto in 60 secondi.

Poi controlla:

- ho definito?
- ho spiegato la funzione?
- ho fatto un esempio?
- ho evitato l'errore tipico?

Se manca uno di questi passaggi, non aggiungere testo. Correggi la scheda.

## Scheda di norma

La scheda di norma non serve a copiare articoli.

Serve a capire come usare un riferimento normativo senza memoria cieca.

Nei concorsi e' utile sapere riconoscere:

- soggetto;
- potere o obbligo;
- presupposto;
- effetto;
- limite;
- collegamento con principi o procedimento.

### Modello

| Campo | Risposta |
|---|---|
| Riferimento | norma, articolo o fonte da verificare |
| Domanda guida | che cosa disciplina? |
| Soggetti | chi agisce o chi subisce effetti |
| Presupposto | quando si applica |
| Effetto | che cosa produce |
| Limite o eccezione | cosa non devo dimenticare |
| Uso in prova | definizione, caso, orale, confronto |
| Errore tipico | |

Questa scheda non sostituisce la fonte.

La rende interrogabile.

Se devi citare un articolo in prova, cita solo cio che conosci con sicurezza. Se non sei sicuro del numero, spiega il principio e la funzione senza inventare.

## Scheda di procedimento

La scheda di procedimento serve quando il tema ha fasi, soggetti, documenti, termini o responsabilita.

E' utile per diritto amministrativo, pubblico impiego, contratti, accesso, controlli, domanda di concorso e gestione dei casi.

### Modello

| Fase | Domanda | Nota |
|---|---|---|
| Avvio | chi o che cosa attiva il procedimento? | |
| Istruttoria | quali elementi vanno acquisiti? | |
| Decisione | chi decide e con quale atto? | |
| Comunicazione | chi deve sapere che cosa? | |
| Controllo | quali rischi o rimedi? | |

Sotto la tabella aggiungi:

| Campo | Risposta |
|---|---|
| Errore tipico | |
| Caso possibile | |
| Domanda da commissario | |

La scheda di procedimento deve farti ragionare come funzionario.

Non basta ricordare le fasi. Devi sapere chi fa che cosa e perche.

## Scheda di confronto

Molte domande-trappola nascono da concetti vicini.

La scheda di confronto serve a evitare confusioni.

Esempi:

- accesso documentale, accesso civico semplice, accesso civico generalizzato;
- annullabilita e nullita;
- organo politico e dirigente;
- quiz con banca dati ufficiale e quiz senza banca dati;
- manuale, corso e banca dati;
- esercizio, test e simulazione.

### Modello

| Voce | Concetto A | Concetto B |
|---|---|---|
| Funzione | | |
| Quando si usa | | |
| Soggetti | | |
| Effetto | | |
| Errore tipico | | |
| Domanda-trappola | | |

Chiudi sempre con una frase:

> la differenza decisiva e'...

Se non riesci a scriverla, la scheda non e' ancora chiara.

## Scheda di domanda orale

La scheda orale non deve essere un copione.

Deve essere una pista.

### Modello

| Campo | Risposta |
|---|---|
| Domanda | |
| Apertura | una frase di inquadramento |
| Definizione | |
| Funzione | |
| Riferimento sicuro | solo se davvero sicuro |
| Esempio | |
| Collegamento | materia, profilo o caso |
| Chiusura | risposta alla domanda |

Allenati in tre formati:

| Formato | Obiettivo |
|---|---|
| 30 secondi | definizione essenziale |
| 2 minuti | risposta standard |
| 5 minuti | collegamenti e caso |

Non imparare la scheda a memoria.

Usala per controllare ordine e completezza.

## Scheda di errore

La scheda di errore nasce dal Diario.

Non la creare per ogni errore.

Creala quando l'errore si ripete, costa punti o rivela una confusione profonda.

### Modello

| Campo | Risposta |
|---|---|
| Errore | che cosa ho sbagliato |
| Causa | memoria, concetto, lettura, fretta, strategia, ansia |
| Regola corretta | |
| Esempio corretto | |
| Segnale di allarme | parola o situazione che mi fa sbagliare |
| Drill | come verifico la correzione |
| Data ritorno | quando la riprovo |

Una scheda di errore deve finire con un drill.

Se finisce con una spiegazione, e' ancora teoria.

## Scheda di caso

La scheda di caso serve a trasformare il sapere in decisione.

Non deve essere lunga.

Deve obbligarti a separare dati, soggetti, vincoli e soluzione.

### Modello

| Campo | Risposta |
|---|---|
| Situazione | fatto essenziale |
| Soggetti | chi e' coinvolto |
| Problema | qual e' la questione |
| Vincoli | termini, dati, competenza, riservatezza, risorse |
| Regola o principio | solo cio che e' sicuro |
| Soluzione | cosa farei |
| Comunicazione | come lo spiegherei |
| Rischio | quale errore evitare |

La scheda di caso e' utile soprattutto per:

- prova teorico-pratica;
- quesiti situazionali;
- orale con esempi;
- problem solving amministrativo;
- simulazioni miste.

## Come nasce una scheda

Non creare schede mentre leggi tutto in modo passivo.

Segui questo percorso.

1. Leggi il bando o il blocco di programma.
2. Scegli un nucleo ad alta resa.
3. Studia la fonte o il manuale.
4. Formula una domanda.
5. Prova a rispondere senza guardare.
6. Scrivi solo cio che serve a migliorare la risposta.
7. Aggiungi errore tipico e verifica.
8. Inserisci la scheda nel piano o nel Diario.

La scheda migliore spesso nasce dopo il primo errore, non prima.

Prima dell'errore pensi di sapere.

Dopo l'errore sai che cosa manca.

## Quando aggiornare una scheda

Aggiorna una scheda se:

- un nuovo bando cambia l'uso del tema;
- una simulazione mostra un errore ricorrente;
- una risposta orale resta disordinata;
- una norma o fonte va verificata;
- hai aggiunto un esempio piu chiaro;
- la scheda e' troppo lunga;
- la scheda non produce piu output.

Non aggiornare una scheda per perfezionismo.

La scheda deve essere abbastanza buona da farti rispondere meglio.

Non deve diventare un manoscritto parallelo.

## Quando archiviare

Archivia una scheda quando:

- duplica un'altra;
- non e' collegata a nessun concorso;
- non ha fonte;
- non ha domanda;
- non ha uso in prova;
- e' vecchia e non verificata;
- la apri spesso ma non la usi mai.

Archiviare non significa perdere.

Significa proteggere attenzione.

Il capitale di studio non e' tutto cio che hai prodotto. E' cio che puoi riusare con fiducia.

## Caso guidato

Luca prepara un concorso amministrativo. Ha sottolineato molte pagine sul procedimento amministrativo, ma quando prova a rispondere oralmente si perde: parte dalla definizione, poi cita dettagli, poi dimentica il collegamento con il caso.

Decide di creare una scheda operativa.

Domanda guida:

> Come spiego il procedimento amministrativo in una risposta orale da due minuti?

Compila:

| Campo | Risposta sintetica |
|---|---|
| Nucleo | sequenza ordinata con cui la PA arriva a una decisione |
| Funzione | rendere l'azione amministrativa controllabile, partecipata e motivata |
| Fasi | avvio, istruttoria, decisione, comunicazione, controlli |
| Esempio | istanza del cittadino a un ufficio comunale |
| Errore tipico | recitare fasi senza spiegare chi fa cosa |
| Verifica | risposta orale da 2 minuti con esempio |

Poi registra una simulazione orale.

Si accorge che parla troppo nella fase di avvio e non chiude la risposta.

Aggiorna la scheda:

- aggiunge una frase di chiusura;
- riduce le fasi a parole chiave;
- inserisce "chi decide?" come domanda di controllo.

Dopo due giorni riprova.

La risposta e' piu breve, piu ordinata e piu vicina alla domanda.

Luca non ha fatto un riassunto migliore.

Ha creato una scheda che corregge un output.

## Da sapere in 5 righe

1. Una scheda operativa non riassume tutto: prepara un output.
2. Ogni scheda deve avere domanda guida, nucleo, fonte, uso in prova, errore e verifica.
3. Il Diario degli errori decide quali schede creare o correggere.
4. Le schede migliori sono brevi, interrogabili e riutilizzabili.
5. Una scheda non verificata con richiamo, risposta o simulazione resta materiale passivo.

## Domanda da commissario

**Domanda:** Perche in una preparazione concorsuale gli appunti non bastano?

**Risposta efficace:** perche gli appunti registrano informazioni, ma non garantiscono che il candidato sappia usarle nel formato della prova. Una scheda operativa trasforma il contenuto in domanda, risposta, esempio, errore da evitare e verifica. In questo modo collega studio, Diario degli errori e output concorsuale: quiz, scritto, orale, caso o simulazione.

## Domanda-trappola

**Domanda:** Se una scheda e' molto completa, e' automaticamente una buona scheda?

**Risposta:** no. Una scheda troppo completa puo diventare un riassunto passivo. Una buona scheda e' proporzionata all'uso: contiene cio che serve per rispondere, correggere un errore o preparare una simulazione. Se non ha domanda guida, errore tipico e verifica, puo essere ordinata ma non operativa.

## Errore tipico

L'errore tipico e' trasformare ogni capitolo in una scheda.

Questo produce molte pagine, ma poco controllo.

Il candidato forte crea schede quando c'e':

- un nucleo ad alta resa;
- un errore ricorrente;
- una domanda probabile;
- una confusione tra concetti;
- una simulazione da migliorare;
- un output da allenare.

Non chiederti: "ho riassunto tutto?"

Chiediti: "questa scheda mi fa rispondere meglio?"

## Mini-esercizio

Scegli un errore recente dal Diario.

Compila una scheda di errore.

| Campo | Risposta |
|---|---|
| Errore commesso | |
| Causa | memoria / concetto / lettura / fretta / strategia / ansia |
| Domanda guida | |
| Regola corretta | |
| Esempio | |
| Errore da evitare | |
| Verifica entro 7 giorni | |

Poi trasformala in un output.

| Output | Prova |
|---|---|
| Quiz | creo 5 domande simili |
| Scritto | scrivo una risposta da 10 righe |
| Orale | rispondo in 2 minuti |
| Caso | applico il concetto a una situazione |
| Simulazione | inserisco il punto nella prossima prova |

La scheda e' finita solo quando l'hai provata.

## Checklist finale

Prima di salvare una scheda nel tuo sistema, controlla:

- ha una domanda guida;
- ha un nucleo centrale;
- indica la fonte o il materiale di partenza;
- spiega l'uso in prova;
- contiene un esempio o mini-applicazione;
- segnala un errore tipico;
- prevede una verifica;
- sta in una pagina o in un blocco breve;
- non duplica una scheda gia esistente;
- puo entrare nel Diario, nel piano o in una simulazione;
- puo essere aggiornata senza riscrivere tutto;
- puo essere riusata in un altro concorso simile.

Se mancano domanda, errore e verifica, non e' ancora una scheda operativa.

E' solo un appunto piu ordinato.
